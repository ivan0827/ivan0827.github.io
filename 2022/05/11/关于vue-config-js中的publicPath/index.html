##关于vue.config.js中的publicPath

####简单理解vue.config.js中的publicPath

使用vue2创建一个默认项目后，在项目根目录下新建一个vue.config.js文件，其中有个配置叫publicPath，如下

其中process.env.NODE_ENV是判断当前处于开发环境还是生产环境，对应不同环境配置值不同。执行npm run build打包后项目根目录生成一个dist文件夹，存放的就是我们这个项目打包后的东西，在本地启一个服务(我在vsode装了一个Live server插件，直接右键点击dist目录下的index.html文件，选择Open with Live Server)会运行在浏览器，打开开发者工具，点击网络可以看到当前请求的资源，点击其中一个静态资源可以看到请求路径包含了我们设置的publicPath的值。如果不设置这个值打开后页面则会报错，找不到对应路径下的资源。

其中’/dist/‘也可以配置为’./‘，这样所有的资源都会被链接为相对路径，这样打出来的包可以被部署在任意路径。
